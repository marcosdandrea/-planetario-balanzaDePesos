Planetario - Balanza de pesos

Manual tecnico para ajustar el archivo .env

Guia breve para personal tecnico del museo. Este documento explica que variables se pueden cambiar y que hace cada una.

1. Que es el archivo .env

El archivo .env contiene parametros de configuracion de la aplicacion. El tecnico puede modificarlo cuando sea necesario para ajustar el comportamiento de la balanza, la conexion serial o algunos tiempos de espera.

Importante: despues de cambiar el .env, se debe reiniciar la aplicacion para que los cambios se apliquen.
No cambiar por prueba y error: si no se conoce el efecto de una variable, modificar un solo valor por vez y anotar el cambio realizado.
Donde esta el archivo segun la instalacion:
  • Aplicacion instalada (uso normal en el museo): el .env esta en la carpeta resources, al lado del ejecutable (por ejemplo ...\Balanza de Pesos\resources\.env). Se edita directamente con el Bloc de notas; no esta empaquetado ni comprimido dentro de la aplicacion.
  • Entorno de desarrollo: el .env esta en la raiz del proyecto.

2. Variables principales

Variable Para que sirve Valor actual recomendado
SCALE_SERIAL_ENABLED Activa o desactiva la lectura de la balanza. true
SCALE_SERIAL_PORT Puerto serial de la balanza. Normalmente se deja en auto. auto
SCALE_SERIAL_BAUD_RATE Velocidad de comunicacion serial. 9600
SCALE_SERIAL_DATA_BITS Bits de datos de la comunicacion. 8
SCALE_SERIAL_STOP_BITS Bits de parada. 1
SCALE_SERIAL_PARITY Paridad de la comunicacion. none
SCALE_SERIAL_POLL_INTERVAL_MS Tiempo entre consultas a la balanza en milisegundos. 40
SCALE_SERIAL_REQUEST_COMMAND Comando que envia la aplicacion a la balanza para pedir lectura. \x05
SCALE_SERIAL_MIN_DELTA Cambio minimo de peso para considerar que hay una lectura nueva. 0.5 kg
SCALE_DEBOUNCING_TIME_MS Tiempo (en milisegundos) que la app espera despues de detectar un cambio de peso antes de mostrarlo en pantalla, para evitar que el numero "salte" mientras el objeto todavia se esta asentando en la balanza. Si llega un nuevo cambio antes de que se cumpla ese tiempo, la espera se reinicia y solo se muestra el ultimo valor ya estable. 1000 ms (0 = desactivado, muestra cada cambio al instante)
SCALE_SERIAL_WACK_WAIT_MS Tiempo maximo de WACK continuo para mostrar Espere... cuando ya hay peso activo. 5000 ms
SCALE_SERIAL_IDLE_WACK_WAIT_MS Tiempo maximo de WACK continuo para mostrar Espere... cuando la balanza esta en 0 kg. 1000 ms

3. Puerto del servidor y modo de prueba

Variable Para que sirve Valor actual recomendado
MAIN_SERVER_PORT Puerto de red interno que usa la aplicacion para comunicar la pantalla del juego con el servidor. No hace falta tocarlo salvo que otro programa de la computadora este usando el mismo puerto. 50123
USE_UI_TESTER Muestra un panel flotante de prueba sobre la pantalla del juego para simular manualmente un peso, sin necesidad de tener la balanza conectada. Es una herramienta de diagnostico tecnico, no para el publico. false
Importante: si se cambia MAIN_SERVER_PORT, no hace falta editar ningun otro archivo: tanto la ventana de la aplicacion como su conexion interna se ajustan solas al nuevo puerto. Evitar solamente numeros muy comunes (como 3000, 80 u 8080), que suelen estar ocupados por otros programas.
Importante: con USE_UI_TESTER=true aparece, sobre la pantalla del juego, un panel visible con un campo para "setear" un peso manualmente. Se recomienda dejarlo en false durante la exhibicion normal y activarlo solo para pruebas puntuales.

4. Elegir en que pantalla abre la aplicacion

Esto no se configura en el .env, sino como parametro al iniciar el ejecutable. Sirve para instalaciones con mas de una pantalla conectada (por ejemplo, una pantalla de exhibicion y un monitor de control aparte).

Para abrir la aplicacion en la pantalla numero 2, hay que agregar --screen=2 al iniciar el ejecutable (la pantalla 1 es la primera que detecta Windows). Desde un acceso directo: abrir Propiedades, e ir al campo "Destino", agregando --screen=2 despues de las comillas de cierre del ejecutable.
Si el numero de pantalla no es valido (por ejemplo se pide la pantalla 3 pero solo hay 2 conectadas), la aplicacion abre automaticamente en la primera pantalla, sin mostrar ningun error.

5. Que hacer si la balanza responde lento

  1. Revisar primero el cableado y el puerto USB/serial.
  2. Confirmar que SCALE_SERIAL_PORT siga en auto o en el puerto correcto.
  3. Si la lectura salta demasiado, no bajar de golpe SCALE_SERIAL_MIN_DELTA; probar ajustes pequenos.
  4. Si la interfaz muestra Espere... muy seguido, revisar los valores de WACK.

6. Reglas de espera

  • Con la balanza en 0 kg, la app espera 1 segundo de WACK continuo antes de mostrar Espere....
  • Con peso mayor a 0.5 kg, la app espera 5 segundos de WACK continuo antes de mostrar Espere....
  • Mientras dura esa espera, la ultima lectura valida se mantiene en pantalla.

7. Procedimiento seguro para cambiar el .env

  1. Cerrar la aplicacion si esta abierta.
  2. Abrir el archivo .env (ver seccion 1 para la ubicacion segun el tipo de instalacion).
  3. Modificar solo la variable necesaria.
  4. Guardar el archivo.
  5. Reiniciar la aplicacion completa.
  6. Probar la balanza y observar el resultado durante unos segundos.

8. Valores de referencia rapidos

  • Lectura demasiado inestable: subir SCALE_SERIAL_MIN_DELTA.
  • La lectura "tiembla" o cambia varias veces antes de asentarse: subir SCALE_DEBOUNCING_TIME_MS (por ejemplo de 1000 a 1500 ms).
  • El peso tarda demasiado en aparecer en pantalla: bajar SCALE_DEBOUNCING_TIME_MS, con cuidado, ya que valores muy bajos pueden volver a mostrar saltos.
  • La app tarda en reaccionar: revisar SCALE_SERIAL_POLL_INTERVAL_MS y la conexion fisica.
  • Muestra Espere... muy pronto en 0 kg: subir SCALE_SERIAL_IDLE_WACK_WAIT_MS.
  • Muestra Espere... demasiado tarde con peso: bajar SCALE_SERIAL_WACK_WAIT_MS con cuidado.
Recomendacion: si un cambio mejora algo pero rompe otra parte, volver al valor anterior antes de probar otro ajuste.

9. Soporte

Si la balanza deja de responder, anotar el valor cambiado, la hora y el comportamiento observado antes de tocar otra variable. Eso facilita el diagnostico tecnico.
